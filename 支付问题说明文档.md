## 三、技术架构

### 3.1 当前支付链路（Flutter → Android SDK）

```
Flutter (alipay_service.dart)
  │
  ├─ Step 1: 调用 Edge Function 获取订单数据（未签名）
  │           GET https://ibffrwevphkkbcfgaift.supabase.co/functions/v1/alipay-order
  │           ?product=oracle&tradeNo=xxx
  │           返回: { orderData: { app_id, method, biz_content, ... }, outTradeNo }
  │
  ├─ Step 2: Flutter 本地 RSA2 签名
  │           - 从 orderData 中取出所有参数，排除 sign 和 sign_type
  │           - 按字母排序拼接成待签名字符串
  │           - 用 pointycastle 做 SHA256withRSA 签名
  │           - 签名结果 base64 编码后加入 orderData
  │
  ├─ Step 3: 组装 orderString
  │           - 将 orderData 中每个 key=value 做 URL 编码
  │           - 用 & 连接成完整 orderString
  │
  └─ Step 4: 调用 Android 原生方法调起支付宝
              MethodChannel: "com.kouming/alipay"
              方法: pay(orderString)
              最终调用: PayTask(activity).payV2(orderString, true)
```

### 3.2 关键文件

| 文件 | 说明 |
|------|------|
| `lib/services/alipay_service.dart` | Flutter 支付服务，本地签名 + SDK 调用 |
| `android/app/src/main/kotlin/.../MainActivity.kt` | Android 原生活动，MethodChannel 入口 |
| `supabase/functions/alipay-order/index.ts` | Edge Function（返回未签名订单数据） |

### 3.3 签名方式

- **算法**：RSA2（SHA256withRSA）
- **私钥格式**：PKCS#1 → 提取 RSA 参数（n, e, d, p, q, dp, dq, qi）硬编码到 Dart
- **签名库**：Flutter `pointycastle` 包
- **密钥状态**：新密钥对已生成（2026-05-06），新公钥已上传支付宝后台，新私钥已写入代码

## 四、已尝试的方案

### 方案 A：Edge Function 签名（❌ 放弃）

签名逻辑放在 Supabase Edge Function（Deno）中。

**失败原因**：
1. Supabase Secrets 存储私钥时 `\n` 换行符处理异常，DER 解码失败
2. ASN.1 解析函数无法处理"长长度编码"，报 `incorrect length for SEQUENCE`
3. Supabase legacy API keys 被禁用，Edge Function 返回 401
4. 多次修复 DER 解析（v1→v2→v3→v4）均失败

### 方案 B：Flutter 本地签名（✅ 已实现）

将 RSA2 签名从 Edge Function 移到 Flutter 端。

**已实现**：
- 从 PKCS#1 私钥文件提取全部 8 个 RSA 参数（n, e, d, p, q, dp, dq, qi）
- 硬编码到 `alipay_service.dart`，用 `pointycastle` 做 SHA256withRSA 签名
- 签名规范：待签名字符串排除 `sign` 和 `sign_type`，按 key 字母排序拼接
- orderString 组装：每个 value 做 URL 编码，`&` 连接
- Edge Function v5 已部署：只返回未签名 orderData + outTradeNo，不做签名

**当前状态**：
- 模拟支付（mockMode=true）✅ 通过 — UI 流程正常
- Edge Function 返回 ✅ 正常（200 + orderData）
- 真实支付（mockMode=false）❌ **调起支付宝失败** — 需要排查原因

## 五、当前核心问题

### 5.1 支付宝 APP 支付标准流程

```
客户端 → 服务端：请求下单
服务端 → 支付宝网关：调用 alipay.trade.app.pay，生成 signedOrderString
服务端 → 客户端：返回 signedOrderString
客户端 → 支付宝 SDK：传入 signedOrderString
支付宝 SDK → 支付宝 APP：调起支付
```

### 5.2 我们的实际做法

由于服务端（Edge Function）无法做 RSA 签名，我们改为：
1. Edge Function 返回**未签名的订单参数**（app_id、method、charset、biz_content 等）
2. Flutter 拿到后**本地 RSA2 签名**，加入 sign 字段
3. 组装成 orderString，调起支付宝 SDK

### 5.3 需要确认的问题

**❓ 问题 1（最关键）：支付宝 APP 支付是否支持客户端本地签名？**

官方文档和示例都是服务端签名后返回 signedOrderString。我们的场景是：
- 服务端只返回未签名订单参数
- 客户端自行 RSA2 签名后组装 orderString
- 然后调 `PayTask.payV2(orderString, true)`

这样做是否可行？签名在客户端完成是否有安全或兼容性问题？

**❓ 问题 2：签名结果是否兼容？**

我们用 Flutter `pointycastle` 做 SHA256withRSA 签名：
- 算法：SHA256WithRSA，OID 1.2.840.113549.1.1.1
- 待签名字符串：排除 sign/sign_type，按 key 字母排序，`key=value` 用 `&` 连接
- 签名结果：base64 编码后作为 sign 值

签名结果格式是否和支付宝 Java SDK 生成的签名一致？有没有已知的不兼容情况？

**❓ 问题 3：APP 支付权限和配置**

- APP 支付功能是否已在后台开启？（AppID: 2021006149691180）
- 支付结果异步通知URL（notify_url）如何配置？当前 Edge Function 未设置 notify_url
- 是否需要签约"APP支付"产品才能使用？

## 六、相关代码

### 6.1 Flutter 签名代码

```dart
// lib/services/alipay_service.dart
static String _rsa2Sign(String message, RSAPrivateKey privateKey) {
  final signer = RSASigner(SHASignDigest.SHA256, 'RSA/ECB/PKCS1Padding');
  signer.init(PrivateKeyParameter<RSAPrivateKey>(privateKey),
    PublicKeyParameter<RSAPublicKey>(privateKey as RSAPublicKey));
  final signature = signer.generateSignature(utf8.encode(message));
  return base64Encode(signature.bytes);
}

// 构建待签名字符串：排除 sign 和 sign_type，按 key 排序
static String _buildSignString(Map<String, dynamic> orderData) {
  final keys = orderData.keys.where((k) => k != 'sign' && k != 'sign_type').toList()..sort();
  return keys.map((k) => '$k=${orderData[k]}').join('&');
}
```

### 6.2 Android MethodChannel

```kotlin
// MainActivity.kt
MethodChannel(flutterEngine.dartExecutor.binaryMessenger, "com.kouming/alipay")
  .setMethodCallHandler { call, result ->
    if (call.method == "pay") {
      val orderString = call.argument<String>("orderString") ?: ""
      val payTask = PayTask(activity)
      val payResult = payTask.payV2(orderString, true)
      result.success(payResult)
    }
  }
```
