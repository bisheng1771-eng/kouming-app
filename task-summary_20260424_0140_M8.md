## 目标
为叩命 Flutter 应用添加本地数据持久化和模拟支付系统

## 关键决策

### 1. 存储方案：SharedPreferences + JSON
- **理由**: 项目已有 shared_preferences 依赖，无需额外安装
- **实现**: AppState 完整序列化到 JSON，每次状态变更自动保存
- **备选**: Hive（更快但需额外依赖），Firebase（需网络+配置）

### 2. 支付方案：模拟支付服务
- **理由**: 真实支付（微信/支付宝/Stripe）需要商户资质和复杂配置
- **实现**: PaymentService 模拟 800ms 延迟后返回成功
- **扩展点**: 预留接口，后续可直接替换为真实 SDK

### 3. 加载体验
- 启动时显示金色 CircularProgressIndicator
- 加载完成后进入主界面

### 状态持久化流程
```dart
// 启动时加载
Future<void> _loadState() async {
  final saved = await _storage.loadAppState();
  if (saved != null) {
    setState(() { _state = saved; _loading = false; });
  } else {
    setState(() { _loading = false; });
  }
}

// 每次变更后保存
Future<void> _persist() async {
  await _storage.saveAppState(_state);
}
```
